SSH创建秘钥

  • 1.以下命令行会在/your_home_path/.ssh目录下创建id_rsa名称的公钥和私钥，命令执行后会提示输入passphrase，可一路回车使用默认设置。

    ssh-keygen -t rsa -C “xx@xx.com“

  • 2.可指定秘钥名称的创建方式

    ssh-keygen -t rsa -f id_rsa_xx -C “xx@xx.com“

  • 3.验证秘钥是否配置成功

    ssh -T <用户名>@
    ssh -T git@github.com

1.Git常用设置

查看当前git库配置信息

配置全局邮箱

状态模式

  • 概述
    属于行为型模式，当一个对象的内在状态改变时对应行为也发生改变

    状态模式的目的是为了把上述一大串if…else…的逻辑给分拆到不同的状态类中，使得将来增加状态容易扩展

  • 实例程序

中介者模式

  • 概述
    属于行为型模式，用一个中介对象来封装一系列的对象交互，中介使得各对象不需要显式地相互引用，降低引用之间的耦合，可以独立地改变他们之间的交互

  • 解析
    中介模式是通过引入一个中介对象，把多边关系变成多个双边关系，从而简化系统组件的交互耦合度

  • 实例程序
    多选框 + 全选 + 反选的复杂组合选择模型

责任链模式

  • 概述
    属于行为型模式，是一种处理请求的模式，它让多个处理器都有机会处理该请求，直到其中某个处理成功为止，责任链模式把多个处理器串成链，然后让请求在链上传递

  • 解析
    责任链模式降低了请求发送者和接收者之间的耦合关系，可以将多个处理器组合在一起，依次处理请求的模式，常用语拦截，预处理等场景下

  • 实例程序
    报销审批流程既符合责任链模式

模板模式

  • 概述
    属于行为型模式，定义一个操作的算法骨架，而将一些步骤延迟到子类中。模板方法使得子类可以不改变算法结构的情况下重新定义算法的实现内容

  • 解析
    模板方法模式很好的体现了面向对象的三大特性，封装、继承、多态。 实际场景中一些通用的逻辑封装在父类中，不同的部分在父类中定义抽象方法在子类中重写对应逻辑

  • 实例程序
    支付SDK的架构，统一的支付接口，底层对接不同的支付公司

命令模式

  • 概述
    属于行为型模式，将一个请求封装为一个对象，使得可以用不同的请求进行参数化，适用于请求排队或者记录请求日志以及支持可撤销的操作

  • 解析
    对命令进行封装，由调用者发起命令请求，接收者执行请求，可大大降低依赖之间的耦合性

  • 实例程序
    去餐厅点餐的流程既符合命令模式，顾客选择一个套餐通过点餐机发起请求命令，后厨通过点餐机响应请求，不同的套餐可看做是不同的命令

解释器模式

  • 概述
    属于行为型模式，给定一种语言，定义它的文法和表示，并定义一个解释器，这个解释器使用该表示来解释语言中的句子

  • 解析
    这和解释型编程语言的解释器有点类似，要根据一段输入转换成一段输出，将不易读的文本转换为易读的文本，将机器不能识别的输入转成二进制机器可读的输出

  • 实例程序

本文字数： 1.7k 阅读时长 ≈ 2 分钟

观察者模式

  • 概述
    属于行为型模式，定义了一种一对多的依赖关系，让多个观察者同时监听某一主题对象。 这个主题对象在状态发生变化时，会通知所有观察者对象，使得他们能够自动更新自己

  • 解析
    当一个对象的改变需要改变其他对象的时候，而且它不知道具体有多少对象有待改变的时候，应该考虑使用观察者模式

  • 实例程序
    微信中关注的公众号，公众号有新的内容或者推送时，会提醒已经订阅该公众号的用户

访问者模式

  • 概述
    属于行为型模式，表示一个作用于某对象结构中的各元素的操作，它使你可以在不改变各元素的类的前提下定义作用于这些元素的新操作

  • 解析
    一种可操作一组对象的操作，它的目的是不改变对象的定义，但允许新增不同的访问者，来定义新的操作

  • 实例程序
    访问者模式个人理解像是向敌方安插卧底的场景，不需要破坏敌方的组织架构，而我方可以通过卧底人员获取到敌方的人员，组织架构，火力配置等信息

备忘录模式

  • 实例程序
    代码管理软件Git的设计既符合备忘录模式，可以查看记录并恢复到某个历史状态
