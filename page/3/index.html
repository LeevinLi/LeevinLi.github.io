Java-线程-sleep与wait区别

发表于 2021-01-08 更新于 2021-02-17 分类于 Java 阅读次数：
本文字数： 742 阅读时长 ≈ 1 分钟

sleep()与wait()区别

  • sleep()简介
    1. sleep()方法用于线程控制，调用后使当前线程进入TIMED_WAITING状态，此状态下会让出CPU的资源并且不释放锁(sleep()与锁没直接关系)
    2. sleep()方法是Thread类中的静态方法，它不能改变对象的机锁，当在一个Synchronized块中调用sleep()方法时，线程虽然休眠了但是对象的机锁并没有被释放，其他线程无法访问这个对象
    3. sleep()休眠时间到达后，当前线程会进入READY状态，等待CUP调度执行

    • 进程：是操作系统进行资源分配的最小单位，是程序在计算机上的一次执行活动，进程内的线程

    • 并发：单位时间内，执行的任务量，如单位时间内CPU执行了多少次任务
