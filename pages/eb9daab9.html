Git常用命令

发表于 2019-10-19 更新于 2022-01-14 分类于 Git 阅读次数：
本文字数： 4k 阅读时长 ≈ 4 分钟

SSH创建秘钥

  • 1.以下命令行会在/your_home_path/.ssh目录下创建id_rsa名称的公钥和私钥，命令执行后会提示输入passphrase，可一路回车使用默认设置。

    ssh-keygen -t rsa -C “xx@xx.com“

  • 2.可指定秘钥名称的创建方式

    ssh-keygen -t rsa -f id_rsa_xx -C “xx@xx.com“

  • 3.验证秘钥是否配置成功

    ssh -T <用户名>@
    ssh -T git@github.com

Git常用命令

1.Git常用设置

查看当前git库配置信息

git config –list

配置全局用户名

git config –global user.name “xx”

配置全局邮箱

git config –global user.email “xx@xx.com“

配置命令自动着色

git config –global color.ui true
git config –global color.status auto
git config –global color.diff auto
git config –global color.diff auto
git config –global color.branch auto
git config –global color.interactive auto

移除代理

git config –global –unset http.proxy

忽略文件权限

git config –add core.filemode false

忽略额外的^M字符

git config –global core.whitespace cr-at-eol

2.新建Git仓库

初始化本地git仓库

git init

当前目录下创建一个文件夹，并初始化为Git仓库

git init xx

从远端克隆一个仓库到本地

git clone git@github.com:LeevinLi/xx.git

克隆远端仓库，并指定本地保存的目录

git clone git@github.com:LeevinLi/xx.git /xx/xx

克隆远端仓库，并指定分支新和本地保存的目录

git clone -b xx git@github.com:LeevinLi/xx.git /xx/xx

3.文件添加/删除/提交操作

添加指定文件到暂缓区

git add xx

添加当前目录所有文件到暂缓区

git add .

删除指定文件，并放入暂缓区

git rm xx

停止跟踪指定文件

git rm –cached xx

修改文件名称并放入暂缓区

git mv oldName newName

指定提交信息，并将修改提交到本地仓库

git commit -m “xx”

将add和commit合为一步

git commit -am “xx”

提交工作区变化直接到本地仓库

git commit -a

提交时显示所有diff信息

git commit -v

合并上一次提交（用于反复修改）

git commit –amend -m “xx”

4.撤销/回退操作

恢复暂缓区的指定文件到工作区

git checkout xx

恢复某个commit的指定文件到暂缓区和工作区

git checkout commitid xx

恢复暂缓区的所有文件到工作区(*工作区修改会丢失)

git checkout .

重置暂缓区的指定文件或commit

git reset xx

重置暂缓区和工作区(*工作区修改会丢失)

git reset –hard

强制重置当前分支的HEAD为指定commit，同时改变暂缓区和工作区(*工作区修改会丢失)

git reset –hard commitid

将当前版本重置为HEAD(*工作区修改会丢失)

git reset –hard HEAD

撤销提交

git revert commitid

5.暂缓区操作

将所有修改存入暂存

git stash

将所有修改存入暂存，并设置tag信息

git stash save “xx”

应用上一次的暂存，并删除上一次暂存记录

git stash pop

查看所有暂存

git stash list

查看指定暂存的详细内容(用下标表示)

git stash show -p stash@{0}

应用指定暂存(用下标表示)

git stash apply stash@{0}

删除指定暂存

git stash drop stash@{0}

6.分支操作

显示所有本地分支

git branch

显示所有远端分支

git branch -r

显示所有分支

git branch -a

新建指定分支，但停留在当前分支

git branch xx

创建新分支，跟踪远端分支，并切换到该分支

git checkout -b local origin/remote
git branch –trach local origin/remote

建立追踪关系，在现有分支与指定的远程分支之间

git branch –set-upstream local origin/remote

切换到指定分支

git checkout xx

切换到上一个分支

git checkout -

新建指向commitid的新分支xx

git branch xx commitid

显示包含提交xx的分支

git branch –contains xx

显示所有已合并到当前分支的分支

git branch –merged

显示所有未合并到当前分支的分支

git branch –no-merged

修改本地分支名称

git branch -m oldName newName

删除指定分支

git branch -d xx

强制删除指定分支

git branch -D xx

图示当前分支历史

git show-branch

图示所有分支历史

git show-branch –all

删除远程分支

git push origin –delete xx
git branch -dr xx

合并xx分支到当前分支(远程分支则使用origin/xx)

git merge xx

合并指定提交至当前分支

git cherry-pick commitid

7.远程仓库操作

下载远端仓库修改

git fetch xx

同步远端仓库修改至本地分支

git pull remote local

显示所有远端仓库

git remote -v

显示指定远端仓库信息

git remote show xx.git

本地已经存在的项目关联指定远端仓库

git remote add origin xx.git

添加远端跟踪仓库地址

git remote add upstream xx.git

修改远程跟踪仓库地址

git remote set-url upstream xx.git

上传本地指定分支到远端仓库

git push remote local

推送当前分支到远程xx分支

git push origin xx

强行推送本地分支到远端分支(本地会覆盖远端)

git push remote –force

推送所有分支到远端仓库

git push remote –all

8.信息查看

查看当前版本状态（是否修改）

git status

查看暂存区和工作区的差异

git diff

查看工作区和分支最新commit的差异

git diff HEAD

查看提交日志

git log

查看最近n次提交日志

git log -5

提交日志及相关变动文件

git log –stat

查看指定文件提交diff记录

git log -p xx

根据关键词查找提交历史

git log -S xx

查看指定文件的版本历史

git log –follow xx

显示所有未添加至index的变更

git diff

显示暂存区和上一个commit的差异

git diff –cached xx

比较与上一个版本的差异

git diff HEAD^

比较两个分支的差异（远程分支origin/master）

git diff xxx…xxx

只显示差异的文件，不显示具体内容

git diff origin/master..master –stat

查看某次提交的详细内容

git show commitid

显示最近一次提交的详细内容

git show HEAD

显示最近几次提交的详细内容(N次就N个^)

git show HEAD^

显示所有用户的提交次数

git shortlog -sn

显示指定文件的修改记录

git blame xx

显示所有提交，包括孤立节点

git reflog

9.patch操作

导出最近的1次commit的patch

git format-patch HEAD^

导出最近的3次commit的patch

git format-patch HEAD^^^

将指定commit变化内容导出为patch文件

git format-patch commit…commit

导出单个commit的patch

git format-patch -1 commit

导出某commit以来的修改patch（不包含该commit）

git format-patch commit

检查指定的patch文件

git apply –stat xx/xx.patch

检查patch文件是否能正常打入，如果没有任何输出，则说明无冲突

git apply –check xx/xx.patch

应用指定patch文件

git am xx/xx.patch

将路径xx/*.patch按照先后顺序打上

git am xx/*.patch

应用失败时，将已经在am过程中打上的patch废弃掉

git am –abort

更多命令可查阅官方文档：Git 官方中文版
